## 二、图像生成优化法则

### 2.1 毒词表 v1.3（新增5个）

v1.2的23个毒词 + 本项目新发现5个：

| 新增毒词 | AI联想 | 替代 |
|---------|--------|------|
| `hair spreading in water like ink` | 头发变墨汁/触手扩散 | `hair drifting gently in water` |
| `blooming/flower` + 任何抽象词 | 莲花出现 | `expanding in organic folds` |
| `crystallization`（无约束时） | 冰晶/钻石/科幻 | `transforming into pearl, nacre layer forming` |
| `miniature landscape inside` | AI无能→乱画 | 接受模糊半透明倒影 |
| `frame cuts off above NOSE` | 仍给全脸 | 不提脸、只描述下颌/嘴唇/光 |

### 2.2 AI硬上限清单 v1.3（新增2个）

| 硬上限 | 表现 | 替代 |
|--------|------|------|
| 泪滴精确定位眼角 | 3轮全在眼中央/睫毛 | 接受睫毛位置 |
| 液体→固体半透明渐变 | 3轮全是完整珍珠或全水滴 | 双分（上水下珠）或接受纯珍珠 |
| 只拍下半脸无眼 | 6轮全是全脸 | 低头+头发遮面+侧面打光 |
| 泪滴内微型折射风景 | 泪滴太小AI无能 | 砍折射 |
| **珍珠内隐约倒影** | **2轮太明显** | **接受半透明=够好** |
| **"水滴=丝绸"视觉隐喻** | **丝绸太实像布料** | **文案解释，图里不硬造** |

**判断规则强化**：同一问题第2轮没改→直接判硬上限→换构图策略。不等第3轮。

### 2.3 分镜类型难度分级（新）

| 难度 | 类型 | 典型轮次 | 策略 |
|------|------|---------|------|
| ⭐ 低 | 抽象/纯材质/纯光 | 1轮 | 无ref直接跑 |
| ⭐⭐ 中 | 简单物品+手/单人全景 | 1-2轮 | 1张候选够 |
| ⭐⭐⭐ 高 | 人物面部/水下/多元素场景 | 2-4轮 | 4张候选 |
| ⭐⭐⭐⭐ 极高 | 面部特写+约束/AI硬上限区 | 3-6轮 | 首轮不加ref，2轮判硬上限 |
| ⭐⭐⭐⭐⭐ 禁区 | 泪滴定位/半脸无眼/微型折射 | — | 换构图，不硬碰 |

### 2.4 省积分升级策略

v1.2已有6招。新增3招：

| 新招 | 省 | 说明 |
|------|-----|------|
| **分镜阶段砍弱镜** | 3-6积分/镜 | 镜5镜6生完图才砍→浪费。应在故事板阶段判断 |
| **抽象镜不加ref** | 防ref污染+省1轮 | 镜10不加ref避免西施漏进去 |
| **角色ref直接当分镜** | 3积分/次 | 镜5=西施背影ref，镜8=侧脸ref |

## 三、视频生成优化法则

### 3.1 Prompt公式（经过16段验证）

```
[分镜图里的静态元素] + [唯一运动元素+方向+速度] + [光照变化方式] + [氛围延续词]
```

**实战模板**：
```
A [主体] against a [背景]. The [元素] [运动] very slowly [方向]. [光线] [变化方式] gently. Static shot, only [X] moves.
```

**关键约束词清单**（每个prompt必含）：
- 运动速度：`very slowly, gently, barely perceptible, like breathing`
- 镜头：`static shot` 或 `camera slowly pushes toward/pulls back`
- 光：`light pulses gently like breathing, light shifts slowly`

**禁止词清单**（除非爆点镜）：
`suddenly, quickly, dramatic, explosive, fast, rapid, abrupt, sharp`

### 3.2 四模选择决策树（新）

```
镜是抽象/纯色/过渡？
  → image2video（简单动作，不需ref）
  
镜有人物+动作？
  → image2video（分镜图作首帧 + 角色ref中等锁身份）
  
镜是核心爆点+精确变形？
  → frames2video（首帧+尾帧锁定，如镜10泪化珍珠）
  
镜需要多元素精确控制？
  → multimodal2video（多ref多图，本项未使用）
```

### 3.3 运镜精准控制

| 需求 | 写法 | 验证在几段上 |
|------|------|------------|
| 固定镜头+物体自转 | `Static shot, only the pearl rotates` | 镜1/11 ✅ |
| 固定镜头+光变化 | `Static shot, only the light breathes` | 镜8/9 ✅ |
| 推镜 | `Camera slowly pushes toward the nacre surface` | 镜2/6/17 ✅ |
| 拉远 | `Camera very slowly pulls back` | 镜16 ✅ |
| 水下漂浮感 | `Static shot, everything in slow underwater motion` | 镜3 ✅ |

**核心发现**：`Static shot` 放在prompt末句 → Seedance最稳定服从。放在开头容易忽略。

## 四、音效生成优化法则

### 4.1 画面锚定技术（珠泪核心创新）

**原则**：每个声音事件必须对应画面上的具体视觉内容。不是"此时应该有水声"，而是"画面上水滴正在从指尖拉长——此时surface tension stretching声"。

**模板**：
```
[时间]. [声音描述]——[英文关键词], [画面锚定：此刻屏幕上正在发生什么].
```

**实例对比**：

| ❌ 悬浮 | ✅ 锚定 |
|--------|--------|
| 第5秒：古筝一个新单音 | 第5秒：古筝一个单音悬在空中——single guzheng note suspended, echoing the water droplet elongating on screen |
| 第7-8秒：silk fabric drifting | 第7-8秒：silk fabric drifting underwater——soft liquid rustle, like the layered white silk flowing across the screen |

### 4.2 比赛/品牌关键词嵌入策略

**错误**：关键词堆在prompt开头/结尾 → 割裂感
**正确**：关键词只出现在"为什么这个声音属于这个画面"的逻辑链中

```
不是："山下湖淡水珍珠"作为独立标签
而是："一颗淡水珍珠——山下湖的声音"作为整段的情绪署名
```

### 4.3 双平台策略（新）

| 项目类型 | 主用 | 备用 | 原因 |
|---------|------|------|------|
| 精确画面同步 | 可灵7段拼接 | Suno整段 | 可灵时间轴描述更精确 |
| 整体氛围配乐 | Suno一次生成 | 可灵拼接 | Suno内生连贯性更好 |
| 核心高潮段 | 两端都跑 | 选优 | 高潮段值得多一次试错 |

## 五、分镜与故事板优化

### 5.1 分镜评审闸门（新增）

**在生图之前增加一道闸门**——镜5镜6的教训：生完图才发现弱。

分镜脚本阶段评审清单：
- [ ] 每个镜是否有不可替代的情绪功能？
- [ ] 相邻镜是否重复同一情绪/信息？
- [ ] 是否有"过渡镜"可以用更短时长或更简单方式替代？
- [ ] 爆点镜是否足够突出（时长/timing/视觉对比）？
- [ ] 故事弧是否完整：起→承→转→爆→合？

### 5.2 镜头时长分布（珠泪实测优化后）

| 类型 | 时长 | 数量 | 占比 |
|------|------|------|------|
| 爆点/核心锚点 | 6s | 2（镜10/镜13） | 12% |
| 关键镜 | 5s | 7 | 41% |
| 标准镜 | 4s | 6 | 35% |
| 过渡镜 | 3s | 1（镜12） | 6% |
| 落版 | 5s | 1（镜18） | 6% |

> 去掉镜5镜6后：16镜 / ~73s。节奏更紧凑。

### 5.3 角色ref图策略

| ref类型 | 生成轮次 | 拍摄角度 | 用途 |
|---------|---------|---------|------|
| 背影 | 1张 | 正背面 | 锁角色身形+服装+光线 |
| 侧脸 | 1张 | 正侧45° | 锁面部特征+肤色+下颌线 |
| 特写（眼/手） | 0（不需） | — | 特写不露全脸，ref无用 |

**关键**：不要多角度ref（正面/背/侧各一张）→ AI confusion。一个角色最多2张ref（背影+侧脸），多余ref导致风格漂移。

## 六、工作流优化

### 6.1 珠泪实际工作流（vs 理想）

```
理想：脚本 → 分镜评审 → 砍弱镜 → 生图 → 选图 → 视频 → 音效 → 剪辑
实际：脚本 → 生图 → 砍弱镜(晚了) → 补图 → 视频 → 音效 → 剪辑
      ↑ 镜5镜6生完图才发现弱 —— 浪费
```

### 6.2 优化后工作流（下部项目用）

```
Phase 1: 脚本创作（1-2小时）
  ├─ 风格/角色/分镜脚本
  ├─ ⭐ 分镜评审闸门（砍弱镜在这里！）
  └─ 等用户确认
        ↓
Phase 1.5: 分镜图（逐镜）
  ├─ 难度分级（低→中→高→极高→禁区）
  ├─ ⭐ 极高/禁区镜：第2轮不变→直接判硬上限→换构图
  ├─ ⭐ 低难度镜批量生成，高难度镜逐张确认
  └─ 等用户选图确认
        ↓
Phase 2A: 视频（逐镜/批量）
  ├─ 成本闸门预报
  ├─ 优先出1-2段测试参考图效果
  ├─ ⭐ 抽象镜批量提交，人物镜逐段提交
  └─ 每段汇报进度
        ↓
Phase 2B: 音效（与视频并行）
  ├─ 对照视频成片写"画面锚定"prompt
  ├─ 可灵7段拼接（精确同步） + Suno整段（备选）
  └─ 导入剪映对齐
        ↓
Phase 3: 后期（剪映）
  ├─ 视频+音效对齐时间轴
  ├─ 段间交叉淡入淡出0.5s
  ├─ 片头片尾（镜18黑底片名）
  └─ 导出
```

### 6.3 并行策略

```
Phase 2A（视频生成）──┬── 同时 ──┬── Phase 3（后期）
                      │          │
Phase 2B（音效prompt写）─┘          │
                                    │
可灵/Suno生成音效（等视频成片后）────┘
```

音效prompt可在视频生成等待期间写好。视频成片后立即对照微调→生成音效。

## 十、比赛文档与文案写作铁律（v2.2新增）

> 来源：报名表三件套重写5+轮，token浪费占比本次会话~30%

### 10.1 文档类型识别（先问：这是写给谁看的？）

| 文档类型 | 受众 | 语气 | 禁止 |
|---------|------|------|------|
| 比赛报名表 | 评委（创作陈述） | 艺术家陈述 | 禁止：轮次/成本/积分/技术指标堆砌 |
| 项目复盘 | 自己/团队（技术报告） | 数据+根因 | 可以：数字、轮次、浪费分析 |
| Skill文件 | AI/自己（操作手册） | 规则+模板 | 禁止：叙事性描述 |
| 给用户的汇报 | 用户（确认+选项） | 选项菜单 | 禁止：长段落、完整文档 |

**核心规则**：下笔前先判断文档类型。比赛文档≠项目文档≠操作手册。语气错了改措辞没用——需要重写整段结构。

### 10.2 比赛文档五条铁律

1. **开头定调**：第一句决定整篇气质。用"一滴西施的眼泪，化为山下湖一颗珍珠"——不用"本作品采用AI技术..."
2. **技术说明=创作选择**：说"为什么选这个工具、这个方式"，不说"用了什么工具、什么参数"。prompt顺序是因为Seedream推理像人眼——不是因为"优化后效率提升"
3. **数字只在有情感意义时出现**："63秒跨越了两千五百年"可以。"18镜×2.8轮"不行
4. **工具清单只列实际使用的**：计划用但最终未用的工具一律不写。可灵写了7段prompt但从未生成→不写
5. **同一反馈出现2次=结构错了**：评委说"像项目报告"→第一次改措辞→第二次还像→停止微调，重新思考整段受众和结构

### 10.3 比赛文档常见错误

| ❌ 错误 | 表现 | ✅ 修正 |
|---------|------|--------|
| 项目交付语气 | "作品同时命中大赛六大创作方向中的三个" | "一滴眼泪穿越两千五百年变成一颗珍珠" |
| 技术指标堆砌 | "18镜×2.8轮、~1030积分" | 不写数字，写创作选择的原因 |
| 工具罗列 | "图像用A、视频用B、音效用C" | 合并为一段："所有分镜图通过...视频用...配乐用..." |
| 功能描述 | "实现了首尾帧锁定模式" | "给AI第一帧和最后一帧，让它自己算出中间的变形过程" |
| 小节标题分段 | "## 图像\n...\n## 视频\n..." | 一个连续段落，无标题 |

### 10.4 工具引用验证规则

项目收尾阶段，所有文档中的工具清单必须过验证：

- [ ] 每个列出的工具确实生成了最终使用的素材？
- [ ] 写了prompt但从未生成的工具→删除
- [ ] "计划用但改用"的工具→不写（除非是创作选择的一部分）
- [ ] 版本号准确？（Seedream 5.0 vs 2.0，Suno v4 vs v5）
