## Mx-Shell 提示词方法论（2026-06-08 丧尸清道夫深度拆解）

> 来源：Mx-Shell（刘梓瑜）本人两段B站分享视频 + 多篇深度专访拆解。
> 3分34秒AI短片《丧尸清道夫》，10天/3000元/一个人，全网播放破亿，好莱坞寻人合作。
> 适用：所有AI视频/图像创作项目。与上方锚点连续流架构互补——技法+心法。

### 核心创作哲学

**"我是导演，AI是演员。不把它定死，它反而会给我超出意料的好画面。"**

| 传统做法 | Mx-Shell做法 |
|----------|-------------|
| 先写完整剧本再开工 | 边生成边修改边推进，做完才有完整剧本 |
| 逐镜画分镜图再出视频 | 一张场景参考图 + 一张角色图 + 一段文字指令 |
| 提示词=动作指令 | 提示词=导演台本（有因果逻辑，不只说做什么） |
| 生成一条满意就停 | "保一条"——已有可用画面后继续多跑几次碰更好版本 |
| 追求技术完美 | 拼的是人的思想和审美，不是工具 |

### 五层提示词结构（从骨架到灵魂）

```
第一层：给AI方向而非模板
"身形修长的人形机器人，1960年代原子朋克风格设计，
面部LED显示屏代替五官显示像素表情"

第二层：用参照物告诉AI"哪种好看"
"模拟IMAX胶片摄影机，搭配Panavision C系列镜头。
复古暖橙加海盐蓝高对比色调。低饱和复古胶片滤镜。
参考美剧《辐射》美学风格"

第三层：几何关系构图
"丧尸头颅位于画面中心分割画面，左侧泳池，右侧走道和躺椅。
景别是贴近地面低机位水平长焦拍摄，运镜是固定机位。
机器人的脚从右侧迅速入画，将头颅踢向左侧"

第四层：时间轴精确到秒
"脚从哪个方向入画 → 头颅飞出去 → 海鸟在哪一秒接力叼走"

第五层：情绪落点
"孤寂的末日孤胆英雄既视感"
```

**每层作用：**
| 层 | 作用 | 缺了会怎样 |
|----|------|-----------|
| 方向层 | 锁定视觉类型，防AI随机发挥 | 画面风格漂移 |
| 参照层 | 告诉AI"审美标准"，非技术参数 | AI味太重 |
| 构图层 | 空间关系+摄影机位置+运镜 | 空间漂移、接缝跳变 |
| 时间层 | 动作顺序+节奏 | 动作崩坏、物理错误 |
| 情绪层 | 气氛落点，观众情感锚 | 画面精美但空洞 |

### "为什么"行为逻辑法则（最核心秘密）

**给AI因果链，不给机械指令。** AI理解角色动机后生成的画面更自然、更有"人味"。

| 错误写法（指令式） | 正确写法（逻辑式） |
|-------------------|-------------------|
| "角色压住帽子" | "角色狂奔时压住帽子，因为怕帽子被风吹掉" |
| "角色在哭" | "角色在哭，因为刚发现伙伴已经变成丧尸" |
| "鸵鸟踩到报纸" | "鸵鸟踩到报纸感到不舒服，单脚跳跃甩腿把报纸甩开" |
| "角色在笑" | "角色在笑，因为她看到草莓酸奶还完好无损" |

**规则**：每个动作描述后接 `因为___` 。告诉AI角色为什么移动、为什么停顿、为什么有情绪。因果逻辑直接影响AI最终生成的状态。

### 风格差异化策略

**不要做赛博朋克，每个人都在做赛博朋克。** 找到一个没人做的细分视觉定位。

```
方法论：
1. 列出该题材所有常见风格 → 全部规避
2. 找小众复古美学（原子朋克/昭和风/水彩绘本/剪纸/皮影/浮世绘）
3. 混搭跨界元素（机器人+西部牛仔+鸵鸟坐骑+塑料模特女友）
4. 用作品对标而非风格标签：《机器人总动员》> "复古未来主义"
```

### 轻量图片资产工作流（不用分镜图）

传统分镜图工作流 = 每镜画图 → 出视频。Mx-Shell的做法完全不同：

```
资产 = 一张角色定妆照 + 一张场景概念图（锁定空间关系）
控制 = 一段文字指令（摄影机位置+角度+运动方向）
理念 = 参考图是锚点不是模板。给AI留发挥空间
```

**场景概念图必须锁定空间关系：**
```
"三面墙空间/室内结构完整，左侧是咖啡机，右侧是水槽，正对后门"
→ 同一场景不同角度3张图必须空间结构一致
```

### 抽卡策略（成本控制）

```
Step 1: 跑出第一个可用版本 → 存档
Step 2: 用同一prompt继续多跑2-3次
Step 3: 从所有版本中选最佳
Step 4: 关键镜头（起/转/爆点）可抽卡5-20次
```

- 全片10天改了200多遍
- **生图便宜（3积分/张），视频贵（11积分/秒）。** 优先在生图阶段多抽卡，把不确定性消耗在廉价环节
- 桥接帧=3积分的保险，永远值得

### 视频2完整学习：图片资产工作流底层技法（2026-06-09 逐字转录分析）

> 来源：BV1qJ7C6cEfS 26分钟完整Whisper转录。视频1（BV1xuVC6AEbg 43分钟）待转录。
> 以下技法是底层设计思路和工作流架构，所有AI图像/视频项目通用。

#### 技法1：多层渐进构建（"地基→一层层往上盖"）

Mx-Shell原话："我不喜欢就是一下就让它定死。平地起高楼也要有个地基然后再一层一层的往上盖。"

**执行规则：**
```
每次生图只锁定一个维度，逐层叠加：

第1轮：锁定角色基础造型（体型/配色/轮廓）
第2轮：在第1轮基础上叠加细节（关节结构/纹理/磨损）
第3轮：叠加表情变体（同一角色不同情绪）
第4轮：叠加道具/服装（武器/衣服/配饰）
第5轮：叠加场景环境

禁止：第一轮就把所有细节写进prompt → AI会全部吃掉但每样都做不好
```

**每轮prompt只新增1-2个约束，其他保持上一轮锁定值。**

#### 技法2：多模型分工（不同阶段用不同模型）

Mx-Shell原文中提到的模型选择逻辑：

| 模型 | 长处 | 弱项 | 适用阶段 |
|------|------|------|----------|
| **Midjourney** | 轻量化/便捷/高效 | 缺少神韵和创造力 | 快速构图/手绘参考/简单场景 |
| **Niji** | 艺术性强/美学张力大/创造力高 | 不可控/中文理解差 | 需要艺术神韵的画面 |
| **Flux Max** | 细节逼真/写实质感强 | 速度慢/价格贵/中文崩坏 | 写实特写/材质细节 |
| **GPT Image 2** | 综合最强：神韵+创造力+精准+稳定 | — | 角色定妆/故事版/最终出图 |

**规则：快速探索阶段用MJ（便宜快），锁定方向后用GPT Image 2出成品。**

#### 技法3："不完美"真实性原则

Mx-Shell原话："越完美的东西它反而越假。真实的世界里什么东西都是有瑕疵的。稍微留一些瑕疵——就是轻微污渍、灰尘、划痕——不要让你的造型太过于闪亮或者说太过于完美，就跟刚出场一样，那样看下去有点假。"

**执行规则（所有项目通用）：**
```
角色prompt必须包含"不完美"要素：
- 头发：微乱/几根翘起/不对称分线
- 服装：自然褶皱/轻微磨损/边角不整齐
- 材质：金属有划痕/布料有纹理不均/皮革有使用痕迹
- 机械：关节处轻微污渍/表面灰尘/螺丝有色差
- 人物：表情不对称（左眉稍高/嘴角微偏）/皮肤有毛孔
```

**禁止词汇（会让AI出"塑料感"画面）：**
`perfect, flawless, pristine, brand new, shiny, polished, symmetrical, immaculate`

#### 技法4："记忆点"分散注意力策略

Mx-Shell原话："在人物身上留下一些记忆点，它可以分散人的注意力。大家在看你的影片的时候可能会被这些记忆点所吸引，而忘记去找你这个视频里出现的穿帮。"

**执行规则：**
```
每个角色至少1个"记忆点"——视觉上显眼但不突兀的细节：

机器人清洁夫 → LED像素表情（你看他的脸，就不会盯着关节看）
塑料模特女友 → 红色连衣裙+金色假发（视觉重心在人偶上半身）
丧尸 → 头部外露脑组织（注意力锚点，身体动作瑕疵被忽略）

仓仓项目对应：
- Q版：超大草莓酸奶杯（比身体还大）→ 注意力锚点
- 人形：粉色水母头双层结构 → 记忆点
- 姐姐：细框圆眼镜 → 记忆点
```

**规则：记忆点必须在角色设计阶段就确定，不是prompt临时加。**

#### 技法5：标注即指令——三通道输入体系（2026-06-09 帧级分析修正）

> Mx-Shell在视频中亲自演示：把手绘草图当画布，用红蓝绿三色标注直接指挥AI生成。
> **标注不只是参考——标注就是空间指令。** 圆圈告诉AI"人物放这里"，箭头告诉AI"朝这个方向移动"。

**三通道输入架构：**

```
通道1: 原始场景概念图 → 视觉锚点（画面长什么样）
通道2: 标注草图（手绘版+标注）→ 空间指令（东西放哪里、怎么动）
通道3: 文字prompt → 行为指令（角色做什么、为什么）
```

**三色标注编码系统（Mx-Shell实际演示的配色）：**

| 颜色 | 标记形式 | 标注文字 | 含义 |
|------|----------|----------|------|
| 🔴 红色 | 矩形框 | （无文字/区域标注） | 画面焦点区域/需要AI重点处理的选区 |
| 🔵 蓝色 | 椭圆圈+箭头 | "人物位置" | 角色在画面中的空间坐标 |
| 🟢 绿色 | 箭头/线条 | "行动路线"/"行动轨迹" | 角色移动方向和路径 |

**底部指令条：** 黄色文字栏写额外约束，如"原始图层必须保留"、"它可以减少算力的占用"。

**完整工作流（帧011-025逐帧还原）：**

```
Step 1: 生成场景概念图（高质量原图）
Step 2: AI将原图转黑白手绘草图（低分辨率，省积分+分离标注层）
Step 3: 在草图上用画笔工具标注：
        - 红框 → 框选画面重点区域
        - 蓝圈+箭头 → 标记"人物位置"（可多个，标记不同角色）
        - 绿箭头+线条 → 画"行动路线"（从哪到哪）
        - 黄底文字条 → 额外约束/备注
Step 4: 上传时两张图分开——
       图A = 原始场景概念图（视觉锚点，不含标注）
       图B = 标注草图（空间指令，含所有红蓝绿标注+文字）
Step 5: 文字prompt描述动作逻辑（"因为___"因果链）
       → AI综合三通道输入生成最终画面
```

**为什么三通道比纯文字更精确：**

```
纯文字: "角色在画面左侧靠近门口的位置" → AI靠语言理解推断空间 → 位置偏差大
标注指令: 蓝圈画在草图上门口的具体像素位置 → AI看到空间坐标 → 位置精确

本质区别：文字描述空间关系（模糊），标注锁定空间坐标（精确）。
```

**标注工具界面（Mx-Shell平台实测）：**
- 工具栏图标：画笔 / 橡皮擦 / 选择框 / 填充 / 放大镜
- 颜色选择器：红色 / 蓝色 / 绿色 / 黄色
- 可拖拽文字标签（写好后拖到对应位置）

**禁止事项：**
- 禁止在原始场景图上直接标注 → AI会把标注画进成图
- 禁止只用一种颜色标注所有信息 → AI无法区分"位置"和"路径"
- 禁止标注文字和prompt文字矛盾 → 两通道信息冲突导致生成失败

#### 技法6：四宫格抽卡+节点式资产管理

Mx-Shell工作流中的平台功能提炼：

```
四宫格策略：
- 一次prompt生成4个变体（4宫格）
- 提取最佳单张→设为主图(main anchor)
- 不满意→基于同一节点重新生成（保留历史）

节点式管理：
- 每个角色/场景=独立节点
- 节点下有：初始生成→修正版→表情变体→道具变体
- 可回溯到任意节点分叉出新方向
- 主图(main anchor)标记当前最佳版本
```

**对非平台用户的替代方案：**
```
文件命名体现节点关系：
仓仓_Q版_正面.png          ← 主图锚点
仓仓_Q版_正面_v2.png       ← 第2版
仓仓_Q版_正面_v2_开心.png  ← v2的表情变体

目录结构=资产库：
output/storyboards/
├── 仓仓/     ← 角色节点
├── 姐姐/     ← 角色节点
├── 场景/     ← 场景节点
└── 参考图/   ← 风格/构图参考
```

#### 技法7：逆向参考法

Mx-Shell用塑料模特照片→AI生成角色pose参考。核心逻辑：

```
正向：文字描述 → AI生成图像（容易AI味）
逆向：真实照片/实物 → AI转风格 → 保留真实感+叠加风格

应用场景：
- 角色pose：真人照片 → AI转二次元 → pose自然不僵硬
- 场景构图：实景照片 → AI转手绘 → 空间关系真实
- 材质参考：实物照片 → AI提取材质 → 质感不飘
```

#### 技法8：场景四字段格式（视频1帧002确认）

Mx-Shell场景描述统一格式，所有项目通用：

```
[景别] 特写/近景/中景/全景/远景
[构图] 主体位置/前景/背景/画面分割方式
[运镜手法] 固定/慢推/横移/跟/低机位/俯拍
[场面内容] 时间秒段+动作+因果关系
```

**示例：**
```
[中景] [机器人位于画面右侧1/3，左侧空间留白] [固定机位低角度仰拍]
[0-3s: 机器人右脚从右侧入画，因为听到左侧有动静突然停住。
 3-6s: 缓缓转头向左，LED表情从冷静切换为警戒红色。]
```

#### 技法9：提示词三段式结构

Mx-Shell所有视频prompt按三段写，不是五层结构（五层是图像prompt用的）：

```
[基础设定] — 时间、地点、人物、关键元素
    ↓ "把参考图给到AI以后对其进行一个简单的描述，
       确保AI模型能够知道我给它的是什么东西，
       同时也能够锁定住我提供的参考图的一致性"

[画质] — 风格核心 + 限制词 + 氛围 + 视觉基调 + 色彩影调
    - 风格核心：需要突出的重点（原子朋克/电影质感/胶片色调...）
    - 限制词：CG感/动作僵硬/过度游戏感 → 用"超写实/极致逼真/真人视频拍摄"对冲
    - 氛围：整体气氛描述（"明亮的场景与恐怖的元素形成强烈反差"）
    - 视觉基调：模拟拍摄设备（IMAX/Panavision/特定镜头型号）
    - 色彩影调：色温/冷暖/饱和度倾向

[画面内容] — 5个子项：
    ① 分镜：单镜还是多镜？每镜时间占比
    ② 景别：特写/近景/中景/全景/史诗大广角/监视器视角
    ③ 构图：三分法/黄金分割/对角线/引导线/荷兰角 + 主体位置+前景+背景
    ④ 运镜手法：推拉摇移/固定/POV/希区柯克变焦
    ⑤ 故事要求：画面呈现的内容+氛围提示词（"整体氛围有默片回写感觉"）
```

**两套结构对应关系：**
| 五层结构（图像prompt） | 三段式（视频prompt） |
|------------------------|---------------------|
| 方向层 | → 基础设定 |
| 参照层 | → 画质·视觉基调 |
| 构图层 | → 画面内容·构图+景别 |
| 时间层 | → 画面内容·分镜 |
| 情绪层 | → 画质·氛围+画面内容·故事要求 |

#### 技法10："不要命令AI，要指引AI"

Mx-Shell原话："不要把AI当作冰冷的工具，要把它想象成你的创作伙伴。不要去命令它，是去指引它来配合你完成每一个复杂的工作。"

**执行规则：**
```
命令式（禁止）：  "角色从左侧走到右侧，拿起杯子喝水"
指引式（正确）：  "角色感到口渴，因为走了很长的路。
                 她注意到桌上有杯水，于是走过去拿起杯子喝了一口。
                 整体氛围轻松日常，像午后阳光透过窗帘。"

区别：命令=只说什么动作。指引=说为什么有这个动作+什么情绪氛围。
```

**"要怎么做 + 为什么这样做 + 需要表达的效果"** — 三段因果链。

#### 技法14：后期剪辑四技法

**① 运动匹配转场（Motion Match Cut）：**
```
前一镜结尾的动作方向 = 后一镜开头的动作方向
例：镜1结尾机器人转身向右 → 镜2开头同样向右继续转身
即使两个镜头不是同一段素材，方向一致=视觉流畅。
```

**② 光向连续性检查：**
```
逐镜检查光源方向（太阳/主光在哪边）
不一致 → 镜像翻转画面使其一致
例：镜1阳光左上方/影子右下 → 镜2影子方向不同 → 水平翻转镜2
```

**③ 速度微调增加力量感：**
```
动作镜头：加速0.1x-0.2x → 速度感/力量感提升
0.1x = 几乎不可察觉但累积效果显著
0.2x = 明显加速感
```

**④ 前景遮罩文字叠加：**
```
原始画面 → 复制一份到上层 → 抠出前景主体 →
文字/字幕放在中层（前景和背景之间）→ 前景遮挡文字产生深度感

结果：文字不是简单叠在画面上，而是"融入"画面空间。
```

#### 技法15：AI辅助写prompt的正确用法

Mx-Shell原话："你实在不懂的才去问AI，但是不能太过依赖它。这样你会失去了自己的一些自主思考。"

**规则：**
```
✅ AI辅助范围：
- 不懂的物理定律（子弹轨迹/爆炸力学）
- 不擅长的专业领域（医学细节/机械结构）
- 难以描述的复杂动作序列

❌ 禁止AI替代：
- 角色的性格/动机（这是你决定的）
- 故事的情感节奏（这是你决定的）
- 视觉风格/色调（这是你决定的）
- 最终prompt审核（AI产出必须人工改写）

流程：自己写初稿 → AI补充专业细节 → 自己改写润色 → 最终审核
```
