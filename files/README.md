## 核心方法论

### 锚点连续流架构（AI视频出片底层框架）

**核心原则：不生成后硬拼，生成前预设衔接。**

```
Step 0: 全片锚点分析（必执行，不可跳过）
  → 逐镜扫描 → 识别跨镜重复元素 → 绘制锚点贯穿图
Step 1: 定义Master Key Frame（基准帧）
  → 提取HSV/坐标/占比/发光/色温
Step 2: 镜头分组（同锚点+同空间+同光源=同组）
Step 3: 逐段出片（prompt嵌入锚点+转场+衔接锁链）
Step 4: 后期最少化（AI转变 > 剪辑转场特效）
```

### Mx-Shell方法论（创作技法）

**来源：《丧尸清道夫》3分34秒AI短片，10天/3000元/一个人，全网播放破亿。**

| 技法 | 核心内容 |
|------|---------|
| 五层提示词 | 方向→参照→构图→时间→情绪 |
| "为什么"因果链 | 每动作接"因为___"（给AI动机非指令） |
| 轻量资产 | 角色图+场景图+文字指令（不画分镜图） |
| 风格差异化 | 避开泛滥风格，找小众复古美学 |
| 抽卡策略 | 保一条→多跑2-3次→关键镜可达20次 |
| 不完美真实 | 瑕疵/污渍/不对称 → 比完美更真实 |
| 三通道输入 | 场景图 + 标注草图 + 文字prompt |

### 提示词设计双轨制

```
角色prompt → 五层结构（细节精确锁定）
场景prompt → 空间框架法（留白给AI，信任大气透视）
```

### F2V防卡顿四闸门（Seedance专属）

| 闸门 | 检查内容 | 超标处理 |
|------|---------|---------|
| 0 跨度预检 | 景别级差/主体位移/动作阶段/背景复杂度 | 拆段+桥接帧 |
| 1 拆段桥梁 | 跨度>2级→拆2段+1桥接帧 | 生图3积分 << 重跑99积分 |
| 2 速度曲线 | prompt必须写恒定速度/缓入缓出 | 防末尾堆积 |
| 3 锚点锁坐标 | 纯色背景锁主体像素位置 | 防微漂 |
| 4 平台参数 | 种子值锁定+2%裁切+10s硬上限 | 跨段一致性 |

## 约定与铁律

### 命名规范

| 类别 | 格式 | 示例 |
|------|------|------|
| 角色图 | `{角色}_{形态}_{视角}.png` | `仓仓_Q版_正面.png` |
| 表情 | `{角色}_表情_{情绪}.png` | `仓仓_表情_开心.png` |
| 场景图 | `{场景}_{视角}.png` | `S1风车原野_主视角.png` |
| 分镜图 | `镜{N}_{描述}.png` | `镜01_西施浣纱.png` |
| 视频 | `镜{N}_{时长}s_{模式}.mp4` | `镜1_9s_f2v.mp4` |

### 提示词输出铁律（不可违反）

每次输出图像/视频prompt必须包含6要素：
1. 正向提示词（完整，可复制粘贴）
2. 负向提示词（完整）
3. 参考图（绝对路径，先ls确认存在）
4. 预期画面（ASCII构图+光源+关键元素+禁止元素）
5. 模型设置（工具/画幅/质量）
6. 保存文件名

### 写入优先铁律

任何产出必须先落盘再开口。禁止仅对话输出prompt/设定/进度变更。

### 数据完整性

汇报进度前必须读最新源文件（ls磁盘 + 读JSON + 读MD），禁止凭记忆。

### GPT Image 2专属规则

- 配饰方位：用画面绝对方位（`LEFT/RIGHT SIDE OF IMAGE`），不用角色左右
- 安全过滤：`barefoot`→`white opaque tights covering feet`
- 防双份：`ONLY ONE bracelet, asymmetrical`
- 人形/Q版prompt隔离（人形禁含chibi词，Q版禁含normal proportions词）

### 即梦 Seedance专属规则

- 仅F2V（首帧+尾帧2图），不超2图
- 单段5-10s，超10s必拆段
- Prompt含速度曲线 + 时间分段
- 锁定种子值 + 处理2%裁切

### 成本速算

| 操作 | 成本 |
|------|------|
| 即梦生图 | ~3积分/张 |
| 即梦F2V视频 | ~11积分/秒 |
| GPT Image 2 | ~$0.04/张 |
| 桥接帧（生图） | ~3积分（= 1/33 重跑视频成本） |
