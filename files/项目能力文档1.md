# AI视频工作流 — 项目能力与效果文档

## 项目概述

AI视频工作流是一套**工业级AI视频制作流水线**，将"创意主题 → 分镜脚本 → 分镜图 → 视频成片"的全流程自动化，同时保留人工创作决策权。三条AI管线协同工作：DeepSeek V4 Pro 负责文字创作、GPT Image 2 负责视觉设计、Seedance 2.0 负责视频生成。

**一句话定位**：输入一个主题，输出一部完整短片——但每一步都由你确认，不做黑盒。

---

## 核心能力

### 1. 智能脚本与分镜设计（DeepSeek V4 Pro）

| 能力 | 说明 |
|------|------|
| 完整剧本生成 | 标题、摘要（≤25字钩子）、故事弧（起承转合）、角色列表、场景列表、道具列表 |
| 工业级分镜表 | 每镜含：时长（4-15s变奏）、景别、运镜、角度、镜头焦段、角色动作、对白（≤15字）、氛围、音效配方 |
| 视觉指纹系统 | 每部作品生成唯一`visual_fingerprint`视觉指纹，贯穿所有生图prompt确保风格统一 |
| 6种画幅支持 | 16:9横屏、9:16竖屏、1:1方形、4:3、3:4、21:9超宽 |
| 角色圣经 | 每角色含：脸型、五官、发型、体型、服装材质（布料纹理/褶皱/垂坠/反光）、色板、辨识特征、角色弧 |
| 场景圣经 | 每场景含：空间描述、光线条件、时间、关键道具、空间布局关系 |
| 道具圣经 | 每道具含：外观、材质、颜色、尺寸、连续性规则 |
| 连续性标记 | 每镜`continuity_flags`追踪服装/发型/道具/光线/天气的连戏状态 |
| 4模预标注 | 每镜自动标注适用的视频生成模式（image2video / frames2video / multiframe2video / multimodal2video） |
| 双语prompt | image_prompt（300-500字英文）+ video_prompt（100-200字英文），含@ref引用语法 |

### 2. 分镜图生成（GPT Image 2）

| 能力 | 说明 |
|------|------|
| 逐镜生图 | 每镜独立生成分镜静态图，作为视频首帧 |
| 候选图策略 | 关键节点（起/转/爆点）生成4张候选、标准镜2张、过渡镜1张 |
| 角色圣经图 | 逐角色生成16:9设计表：上部4角度全身+下部4角度面部特写 |
| 故事板网格 | 多镜合并为单张网格图（≤9s用3×3 / ≤12s用4×3 / >12s用4×4），面板含编号+时长标签 |
| 视觉指纹追加 | 每张图prompt自动追加视觉指纹，确保全局风格统一 |
| 6画幅适配 | 根据画幅比例自动选择输出尺寸（1792×1024 / 1024×1792 等） |
| 本地持久化 | 所有分镜图自动下载到`output/storyboards/`目录 |

### 3. 视频生成（Seedance 2.0）

| 能力 | 说明 |
|------|------|
| 4模自动选择 | image2video（默认80%）→ frames2video（爆点10%）→ multiframe2video（复杂动作5%）→ multimodal2video（兜底5%） |
| 图生视频 | 分镜静态图作为首帧，AI生成后续运动画面 |
| 首尾帧锁定 | 关键爆点镜头同时提供首帧+尾帧，AI插值中间帧 |
| 多帧一镜到底 | 2-20个关键帧连续生成，适合复杂动作长镜头 |
| 多ref综合 | 最多8个参考图融合生成，适合作角色一致性要求高的场景 |
| 双API通道 | 支持xskill.ai和火山引擎Ark两条Seedance API通道 |
| 轮询等待 | 自动轮询任务状态直到完成或超时（最长600s） |
| 批量出片 | `batch_video.py` / `batch_video_v2.py` 支持15镜批量生成+图床上传+下载 |
| 本地持久化 | 所有视频自动下载到`output/videos/`目录 |

### 4. 工作流编排

| 能力 | 说明 |
|------|------|
| 完整流水线 | `workflow.py` 一键串联脚本→分镜图→视频全流程 |
| 断点续传 | 从已有脚本JSON恢复，可选从Phase 1.5（分镜图）或Phase 2（视频）继续 |
| 分阶段执行 | 支持`--skip-images`（只出脚本）、`--skip-video`（只出脚本+图） |
| 成本估算 | 每阶段完成自动打印成本估算 |
| 交互闸门 | CLAUDE.md定义了7个确认闸门，每阶段必须用户确认后才进入下一阶段 |

## 技术架构

```
用户输入主题
      │
      ▼
┌─────────────────────────────────┐
│  Phase 1: 脚本创作               │
│  DeepSeek V4 Pro                │
│  reasoning_effort=high          │
│  thinking=enabled               │
│  → 完整分镜脚本JSON              │
└──────────────┬──────────────────┘
               │ 闸门1-3: 用户确认
               ▼
┌─────────────────────────────────┐
│  Phase 1.5: 分镜图层（核心）      │
│  GPT Image 2                    │
│  角色圣经图 + 逐镜分镜图          │
│  修改成本 = 1/18 视频修改成本     │
└──────────────┬──────────────────┘
               │ 闸门4-5: 用户确认
               ▼
┌─────────────────────────────────┐
│  Phase 2: 视频出片               │
│  Seedance 2.0 (4模auto)         │
│  图→视频，逐镜生成               │
└──────────────┬──────────────────┘
               │ 闸门6-7: 用户确认
               ▼
         输出: MP4视频
```

### 成本参考

| 环节 | 单价 | 15镜项目估算 |
|------|------|-------------|
| 脚本生成 | ~$0.005 | ~$0.005 |
| 分镜图 | ~$0.02-0.05/张 | ~$0.75-1.15 (含角色圣经) |
| 视频生成 | 3-55积分/段 | 按平台定价 |
| 修改分镜图 vs 重出视频 | 1:18 成本比 | 先改图再出片可省50%+ |

## 与传统流程对比

| 维度 | 传统视频制作 | 本工作流 |
|------|------------|---------|
| 脚本到成片周期 | 数天~数周 | 数十分钟~数小时 |
| 单镜修改成本 | 重拍/重渲染（高） | 改prompt重生成（3积分） |
| 角色一致性 | 靠美术组手绘规范 | 角色圣经+视觉指纹AI保证 |
| 多风格尝试 | 每种风格需独立制作 | 改style参数即可重出 |
| 人力需求 | 导演+编剧+美术+动画+后期 | 1人 + AI |
| 输出质量 | 专业可控 | AI级，需人工筛选候选 |

## 使用方式

```bash
# 完整流程
python workflow.py "你的视频主题"

# 指定参数
python workflow.py "赛博朋克夜景" --shots 8 --duration 60 --aspect-ratio 16:9

# 竖屏短剧
python workflow.py "古风修仙战斗" --aspect-ratio 9:16 --shots 12 --grid

# 断点续传（从已有脚本继续出图+视频）
python workflow.py --script output/scripts/xxx.json --from-phase 1.5

# 只出脚本（跳过分镜图和视频）
python workflow.py "主题" --skip-images --skip-video

# 批量出片（已有分镜图）
python batch_video_v2.py
```
