## 一、核心架构（全局CLAUDE.md提取）

## AI视频出片方法论（锚点连续流架构）

> 适用于所有AI视频项目。来源：Chain Continuity Method (UTRGV 2025) + FilmWeaver (AAAI 2026) + CineTrans (ICLR 2026) + Seedance实战。

### 核心原则

**不生成后硬拼，生成前预设衔接。** 锁定锚点 → 派生镜头 → 邻段尾帧=下段首帧 → 零过渡拼贴。

### 五步出片法

**Step 0: 全片锚点分析（新增，必执行）**
- 读完整脚本 → 逐镜扫描所有视觉元素
- 识别跨至少2镜重复出现的元素 = 锚点
- 锚点数量由叙事决定（1到N个），不预设
- 绘制锚点贯穿图 → 识别交汇点
- 输出锚点清单写入项目元数据
- 用户确认后才进入Step 1
- **禁止跳过此步直接写单镜prompt** — 单镜视角天然碎片化

**Step 1: 定义Master Key Frame**
- 锁定一张基准帧作为全片视觉锚点
- 提取硬参数：主体HSV/坐标/占比/发光/色温
- 每个镜头prompt必须引用锚点参数

**Step 2: 镜头分组**
- 同锚点+同空间+同光源 → 同组连续流（AI内完成转变）
- 时空大跳 → 新组，预设转场方式
- 组内零过渡，组间人工转场点 ≤ 总衔接点1/3

**Step 3: 逐段出片（prompt嵌入三要素）**
- 锚点锁定 — Master Key Frame参数
- 转场逻辑 — 本段结尾=下段首帧
- 衔接锁链 — 标注↑上一段结尾→本段→↓下一段开头

**Step 4: 后期最少化**
- AI生成的转变 > 剪辑软件的转场特效
- 只有刻意反差/时空大跳才用硬切

### 3种AI原生转场方式
| 方式 | 适用 | 实现 |
|------|------|------|
| 尾帧锁定 | 同空间连续动作 | F2V尾帧=下一镜首帧 |
| 背景渐变 | 时空穿越/回忆 | I2V prompt描述背景色过渡 |
| 匹配转场 | 形状/颜色/动作相似 | prompt描述相似元素衔接 |

### 全F2V架构（不可违反）

> **所有AI镜头一律F2V。I2V仅当下一镜是后期制作且无首帧图时允许。**
> 硬切/淡入黑 = 剪辑阶段剪掉F2V尾帧过渡帧 → 两镜精确对齐。不是用I2V随机尾帧代替。

```
所有AI镜 →  F2V
  首帧 = 本镜分镜图
  尾帧 = 下一镜首帧分镜图（硬锁定）

唯一例外：下一镜是后期制作且无图 → I2V

跨度太大（首尾帧全画面替换） ≠ 降级I2V
  = 生桥接分镜图（3积分/张）+ 拆成多段F2V
  桥接帧 = 首尾帧中间态 → 前半段尾帧 = 后半段首帧
  生图便宜，视频贵。用廉价生图铺路。
```

### F2V防卡顿四闸门（2026-06-10 珠泪全片复盘，所有项目生效）

> **Seedance F2V不是运动生成，是像素morphing。** 首尾帧差异越大→morphing越暴力→卡顿。
> 详细规则见项目 `CLAUDE.md` Phase 2。此处为全局速查。

```
闸门0: 跨度预检（出片前必执行，比写prompt更优先）
  景别跨度>2级 → 必须拆段
  动作阶段>2个 → 必须拆段
  纯色背景 → 锁坐标+max 8s

闸门1: 拆段+桥接帧（跨度超标的唯一解法）
  生图3积分 << 重跑视频99积分
  桥接帧 = 中间态画面，让AI每段只做1/N的工作

闸门2: prompt速度曲线（必写）
  "ABSOLUTELY CONSTANT VELOCITY — no acceleration, no deceleration"
  "Motion must NOT accumulate in the final second"

闸门3: 锚点锁坐标（纯色背景必备）
  "CENTER POINT at exact same pixel coordinates. Does NOT translate/rotate/drift."

闸门4: 即梦平台参数锁定（每段必做）
  ① 种子值锁定 — 第一段Seed复制→后续全粘同一值。锁风格/人物，不锁动作。
  ② 2%裁切 — Seedance自动放大2%。尾帧→剪映缩小2%→导出→作下一段首帧。
  ③ 单段10s硬上限 — 超10s记忆丢失。拆段后每段5-6s最优。

闸门5: 跨度-时长-中间帧对照
  1级跨度(特→近): 6-8s, 0桥接帧
  2级跨度(近→全): 拆2段各4-6s, 1桥接帧
  3动作阶段: 拆2段, 1桥接帧
  纯色背景: max 8s + 锁坐标

卡顿不在后期解决。后期只能剪接，不能修复morphing变形。
```

### Seedance提示词硬规则（即梦网页端，所有项目生效）

**图生视频硬限制：**
- 仅支持固定镜头/稳定画面
- 单人出镜为佳，多人→ID漂移
- 人物近景或特写，面部≥画面1/3
- 背景纯色或极简，无文字/装饰
- 无手持物品，无身体遮挡

**环境微动法则（I2V核心）：**
- 不添加原图不存在的实体（飞鸟/落叶等）
- 描述"动态"和"变化"，不做加法
- 激活环境粒子：水面波纹/光斑明暗/微风拂叶/微尘游动
- 运镜只用1种：缓慢推镜/平稳横移/固定镜头

**动作描述：**
- 抽象→具象：悲伤=低头/肩颤/眼眶红，喜悦=嘴角上扬/眉眼舒展
- 优先低缓连续小动作，避免高爆发动作
- 秒数分段：`0-3s: ...  3-6s: ...`（模型时间精度不稳，仅作参考）

**每镜必加约束：** `保持无字幕、不要生成Logo、不要生成水印、不要生成文字`

**隐喻陷阱（镜3实测教训）：**
- AI将比喻词字面化执行。`如花瓣绽放`→生成花瓣。`像丝线编织`→生成丝线。
- 禁用植物/动物比喻：绽放/blooming/花瓣/petals/开花/生长/树枝/藤蔓
- 改用矿物/物理术语：沉积/结晶/凝聚/层叠
- 抽象过程→用"转变"描述，不给视觉参照物

**七大翻车及救法：**

| 问题 | 救法 |
|------|------|
| 人物ID漂移 | 人脸特写+全身照双上传 |
| 双胞胎 | 末尾加"禁止生成双胞胎效果" |
| 风格漂移 | 明确约束：2D日漫/3D国风/电影质感 |
| 接缝跳变 | 前段末删6帧，后段首删1帧 |
| 画质劣化 | 控制续写次数，白模视频作输入 |
| 无声音 | 提示词写环境音描述 |
| 字幕/Logo | 加负面约束词 |

### 出片信息完整包（每段必须）
1. 成本(积分)
2. 参考图列表(含绝对路径)
3. 视频提示词(含锚点+转场)
4. 预期效果+衔接锁链(上段结尾→本段→下段开头)
5. 即梦设置
6. 保存文件名

## 全片锚点分析铁律（Phase 2 第零步，任何项目必执行）

> **禁止跳过此步直接写单镜prompt。** 单镜视角天然碎片化——只能看到当前镜头，无法感知跨镜重复元素。跳过此步=锚点遗漏=衔接断裂。

### 锚点定义

**锚点 = 跨至少2个镜头重复出现的任何视觉/叙事元素。** 包括物体、身体部位、光源色温、角色、构图规律、材质。

**锚点数量由叙事决定，不预设。** 1个锚点贯穿全片（极简风）到N个锚点交织（多线叙事）皆可。

### 执行步骤

```
Step 0A: 读完整脚本 → 逐镜列出所有视觉元素
Step 0B: 标注跨镜重复元素 → 标记每个元素在哪些镜出现
Step 0C: 合并同类 → 每个锚点=一个贯穿线
Step 0D: 绘制锚点贯穿图 → 每条锚线从哪个镜到哪个镜
Step 0E: 识别交汇点 → 哪些锚点在哪个镜头交汇（全片情感/叙事最高点）
Step 0F: 输出锚点清单 → 写入项目元数据，作为后续所有prompt强制引用源
```

### 锚点清单模板

```json
{
  "anchor_analysis": {
    "anchors": [
      {
        "id": "A1",
        "name": "珍珠",
        "type": "物体",
        "appears_in": [0, 3, 4, 5, 6],
        "first_appearance": 0,
        "anchor_shot": 3,
        "crosses_with": ["A2_手"],
        "cross_point": "镜6(珍珠落掌心)"
      }
    ],
    "orphan_elements": ["片名字幕(仅镜9)", "序章波纹(仅镜0)"]
  }
}
```

### 输出检查

Step 0A-0F完成后必须向用户展示：
1. 锚点总数 + 每条锚线的镜头覆盖
2. 锚点交汇点（全片叙事最高点）
3. "孤儿元素"（仅出现1次，无需锚点追踪但需注明）

**用户确认锚点清单后，才进入Phase 2闸门6成本预估。**

## Phase 0: 风格差异化选型（新增 — 任何项目必执行）

> 来源：Mx-Shell《丧尸清道夫》核心方法论。**先选风格，再写剧本。** 风格决定辨识度。

### 为什么需要Phase 0

AI视频最大的陷阱：每个人都在用同样的工具，生成同样的画面。赛博朋克霓虹、废土末日、仙侠玄幻——套路化=泯然众人。

**风格差异化 = 全片最核心的创作决策。** 选错了后面全白做。

### 执行步骤

```
Step 0A: 列出该题材的常见风格 → 全部标为"规避"
Step 0B: 搜索小众复古美学/跨界混搭（原子朋克/昭和风/水彩绘本/剪纸/皮影/浮世绘）
Step 0C: 提出2-3个差异化风格方案 + 作品对标
Step 0D: 用户选定后，锁定风格锚点（光线结构/镜头语言/色彩分布/材质规则）
Step 0E: 生成一张"风格锚点图"作为全片视觉基准
```

### 风格锚点四要素（锁定后不可变）

```
├─ 光线结构：侧逆光 / 高反差 / 冷色阴影 或 柔和散射 / 暖调
├─ 镜头语言：中长焦 / 手持微抖 / 低机位 或 广角 / 稳定 / 平视
├─ 色彩分布：青橙分离低饱和 / 粉白柔和 / 高对比冷灰
└─ 材质规则：皮肤偏粗糙 / 金属冷反射 / 布料哑光
```

### 示例

```
普通选择：丧尸末日 → 赛博朋克霓虹（泛滥）
差异化选择：丧尸末日 → 原子朋克+西部牛仔+机器人主角（独特）

《丧尸清道夫》风格公式 = 原子朋克 × 西部牛仔 × 机器人 × 复古胶片质感
对标作品：《机器人总动员》而非《银翼杀手》
```

### 脚本创作模式

**模式A：标准模式（完整剧本先行）**

**模式B：探索模式（Mx-Shell式，边做边改）**

## 提示词设计双轨制（2026-06-12 底层架构·所有项目通用）

> 旧：所有prompt统一五层结构。新：按类型分轨。五层用于角色（需锁定细节），空间框架法用于场景（需留白给AI）。
> 来源：男鹿和雄背景美术技法 + 动画色彩设计 + 8源深度研究。

```
┌─ 角色prompt → 五层结构（不变）
│   方向→参照→构图→时间→情绪。细节精确。配饰/服装/体型/表情必须锁定。
│
└─ 场景prompt → 空间框架法（新）
    ① 空间分层（前景/中景/远景·密度递减）
    ② 巨构锚点（材质/比例/位置·只此三要素）
    ③ 色彩脚本（70/20/10·引用项目色彩表）
    ④ 光源+时间（方向+时段·一行）
    ⑤ 技法词（atmospheric perspective/wet-into-wet/living brushstrokes/luminous haze）
    ⑥ 情绪落点（一个词）
    
    禁止写入：远景细节、云朵形状、每朵花位置、颜色渐变方向、花纹描述（远景不可见=省略）
    信任：AI的大气透视 > 文字描述。AI的色彩感 > 文字约束。
```

**空间框架法模板：**
```
[画风] Ghibli background art style.
[光+时] [方向] light, [时段].
[空间] FOREGROUND: [2-3个具体元素]. MIDGROUND: [1-2元素+空间关系]. BACKGROUND: [巨构+材质+比例·笼统].
[色彩] [主色]70%, [辅色]20%, [点缀]10%.
[技法] atmospheric perspective, wet-into-wet blending, living brushstrokes, luminous haze.
[禁止] No characters. No animals. Pure landscape.
```

## Phase 1.5: 分镜图层 — 交互细则

### 两条路径（根据项目选择）

**路径A：标准分镜图（锚点连续流）**
- 逐镜画完整分镜图，首帧+尾帧双图
- 适用：叙事精确、F2V首尾帧硬锁定需求高

**路径B：轻量资产模式（Mx-Shell式，推荐优先尝试）**
- 只生成：角色多角度视图 + 场景概念图 + 风格锚点图
- **不画分镜图。** 用角色图+场景图+文字指令直接驱动视频生成
- 适用：视觉氛围片、角色IP展示、实验性短片
- 优势：省大量生图积分，AI自由发挥空间大

### 角色圣经图

逐角色生成，每生成一张展示给用户确认：
- 脸型对吗？发型对吗？
- 服装颜色/款式对吗？
- 整体气质对吗？

不对→改prompt重生。OK→下一个角色。

**通用要求（两类路径都需要）：**
- 正面全身 + 面部特写 + 背面 + 3/4侧面（至少4张）
- 表情包（开心/哭/生气/呆/害羞，至少5张）
- Q版双形态（如有）各出正背面

### 场景概念图（轻量模式专用）

如果走路径B，场景概念图比角色图更能决定视频质量：

**必须锁定空间关系：**
```
"三面墙空间，左侧[物体]，右侧[物体]，正对[物体]"
→ 同场景不同角度的图必须空间结构一致
```

**每个场景至少2张：** 主视角 + 反打视角，确保AI理解空间深度。

### 分镜图（标准模式）

**每镜发prompt时必须附带：**
1. 正向提示词（完整，含所有关键元素，按五层结构写）
2. 负向提示词
3. **预期画面描述**（ASCII构图+光源标注+关键元素位置+禁止元素）
4. **行为逻辑**（角色的"为什么"——不只是做什么动作）

告知成本后逐张或批量生成：
- 关键镜（起/转/爆点）生成4张候选，让用户选最佳
- 标准镜生成2张候选
- 过渡镜直接生成1张

每批完成后用户描述画面（或截图），AI逐条对比预期画面给出判断。

### 选图判断标准

用户无法截图时，AI根据预期画面给判断清单：
- 构图对吗？（主体位置、裁切边界）
- 光源对吗？（方向、颜色、双光源交汇）
- 关键元素在吗？（列出该镜必须出现的元素）
- 多了什么不该有的？（列出该镜禁止的元素）
- 有"人味"吗？（角色动作有因果关系，不机械）

## F2V防卡顿铁律（2026-06-10 珠泪全片复盘 — 一劳永逸）

> **Seedance F2V不是运动生成，是像素morphing。** 两张静态图→像素空间插值→中间帧是"算出来的渐变"而非"物理驱动的运动"。首尾帧差异越大→morphing越暴力→卡顿越明显。
>
> 以下规则所有F2V项目自动生效。不可跳过。

### 四类卡顿根因

| 类型 | 根因 | 症状 | 触发条件 |
|------|------|------|----------|
| **挤末段** | AI hold首帧过久→最后1-2s冲刺完成所有变化 | 开头静止太久→结尾突然加速跳变 | 运动量大+时长不足 |
| **跨度崩坏** | 首尾帧空间/构图差异超过AI插值能力 | 中间帧变形/撕裂/元素错位 | 景别跨2级以上（近景→全景） |
| **空洞漂移** | 纯色/极简背景无参照点，AI无法判断运动幅度 | 主体微抖/位置跳动 | 纯白/纯黑/单色背景 |
| **速度堆积** | prompt只说"做什么"不说"多快做" | 动作时快时慢、非线性 | 所有不写速度曲线的prompt |

### 闸门0：跨度预检（出片前必执行，比写prompt更优先）

```
对每个F2V镜头，检查4项硬指标。任一超标→必须拆段，不可跳过。

指标1: 景别跨度
  特写→近景 ✅ OK（1级）
  近景→中景 ✅ OK（1级）
  特写→中景 ⚠️ 风险（2级）→ 加桥接帧
  近景→全景 ❌ 超标（2-3级）→ 必须拆段+桥接帧
  特写→全景 ❌ 严重超标（3级+）→ 必须拆2段+2桥接帧

指标2: 构图主体位移
  主体位置不变（居中→居中）✅ OK
  主体偏移≤画面30%（居中→左1/3）⚠️ 风险→prompt锁定位移路径
  主体偏移>画面30% ❌ 超标→拆段

指标3: 单段动作阶段数
  1个动作阶段 ✅ OK
  2个动作阶段 ⚠️ 风险→速度曲线必写
  3个+动作阶段 ❌ 超标→拆段（每段≤2阶段）

指标4: 背景复杂度
  有纹理/结构/物体参照 ✅ OK
  渐变背景（有方向性）⚠️ 风险→锚点锁坐标
  纯色/极简（无参照物）❌ 高风险→锚点锁坐标+速度曲线+降时长
```

### 闸门1：拆段+桥接帧（跨度超标的唯一解法）

```
核心逻辑：生图便宜(3积分)，视频贵(11积分/秒)。
用廉价生图铺路→把大跨度F2V拆成小跨度多段→每段AI只需做1/N的工作。

拆段公式：
  原F2V: 首帧A → 尾帧B（跨度超标）
  
  修复:
  段1: A → 桥接帧M（5-6s，小跨度）
  段2: M → B（4-5s，小跨度）
  
  桥接帧M = 介于A和B之间的中间态画面
  成本: 1张生图(3积分) vs 重跑视频(99积分) = 1:33

极简桥接帧prompt模板：
  "A frame exactly halfway between [A描述] and [B描述].
   [主体] at [中间位置], [背景] at [中间状态].
   Same style, same lighting direction, same color temperature as both endpoints.
   9:16 vertical. No new elements."
```

### 闸门2：prompt速度曲线（必写，不可省略）

```
每个F2V prompt必须包含以下速度指令之一：

恒定速度（最简单，防堆积）:
  "Motion at ABSOLUTELY CONSTANT VELOCITY throughout — 
   no acceleration, no deceleration, uniform speed from first frame to last.
   The [主体] must NOT pause, NOT speed up, NOT slow down."

缓入缓出（自然动作）:
  "0-1s: motion begins with smooth ease-in (acceleration).
   1-[N-1]s: constant velocity.
   [N-1]-[N]s: gentle ease-out to final position."

仅缓出（防开头跳变，最推荐）:
  "First 1s: motion starts immediately at full speed (no easing in).
   Middle: constant velocity throughout.
   Last 1s: smooth deceleration to natural stop.
   CRITICAL: motion must NOT accumulate in the final second."

防停顿指令（多阶段镜头）:
  "Transitions between phases must be CONTINUOUS — 
   no pause, no freeze, no hold between [阶段1] and [阶段2].
   Motion flows seamlessly from one phase into the next."
```

### 闸门3：锚点锁坐标（纯色背景必备）

```
纯白/纯黑/单色背景 → AI无参照物 → 主体必然微漂。

锁定指令（直接写入时间层）:
  "[主体]'s CENTER POINT must remain at EXACT SAME pixel coordinates
   (dead center of frame) throughout all [N] seconds.
   [主体] does NOT translate, rotate, or drift.
   Only [可变的属性] changes — position and scale stay locked."

禁止词（会让AI困惑）:
  "floating" / "hanging" / "drifting" → AI理解为"可以移动"
  替代: "fixed at center" / "locked in position" / "stationary"
```

### 闸门4：即梦平台参数锁定（Seedance专属，每条必做）

```
这三条是即梦Seedance网页端的硬bug/硬限制。不是prompt能解决的。

① 种子值(Seed)锁定：
  生成第一段→点击生成详情→复制Seed数字→后续所有段粘贴同一值。
  只锁画风/人物特征/光影风格，不锁动作。
  不锁Seed→同角色跨段变脸/色调跳变→拼接必出戏。

② SD2.0 2%裁切（LINUX DO社区多人验证）：
  Seedance自动将画面放大裁切约2%。
  尾帧→用于下一段首帧前→必须在剪映中缩小2%留黑边→导出静帧→上传。
  不处理→每段衔接有2%错位→累积后画面明显偏移→卡顿放大。
  
  操作: 剪映导入尾帧图 → 缩放98% → 导出PNG → 作为下一段首帧上传。

③ 单段10s硬上限（社区实测）：
  超10s→模型记忆丢失概率急剧上升→逻辑断裂/角色漂移。
  超15s→90%+翻车率。
  
  单段: 5-10s
  拆段后每段: 5-6s最优
  过渡镜: 4s（不受此限）
  纯色背景: max 8s（不受此限）
```

### 闸门5：跨度-时长-中间帧 三联对照表

```
| 跨度 | 最小时长 | 中间帧 | 速度曲线 | 典型场景 |
|------|---------|--------|---------|---------|
| 1级(特→近) | 6s | 0张 | 恒定 | 眼角→半脸 |
| 1级(近→中) | 8s | 0张 | 缓出 | 半身→全身 |
| 2级(近→全) | 5s+5s | 1张桥接 | 两段各恒定 | 面部→湖全景 |
| 2级(色域跳) | 5s+4s | 1张桥接 | 两段各恒定 | 蓝绿→琥珀 |
| 3动作阶段 | 4s+5s | 1张桥接 | 段间连续 | 浣纱→停手→回头 |
| 纯色背景 | 8s max | 0张 | 恒定+锁坐标 | 纯白空间转化 |

注意：所有单段≤10s。表中超过10s的=拆段后总和，非单段。
```
